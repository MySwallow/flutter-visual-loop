Build me an autonomous Flutter visual-loop. Given a design-source URL (Figma frame URL or Mockplus develop page URL) and a target Flutter route, the agent should: (1) fetch design data — for Figma call the figma-context MCP, for Mockplus invoke the `mockplus-context` skill via the Skill tool and consume its JSON output plus assets it downloads to a local directory, (2) call `flutter-wright` to lock device size/density and navigate to the route, (3) call `flutter-wright` to capture an on-device screenshot, (4) use vision to compare rendered-vs-design and produce a structured diff report (spacing off by Xpx, color mismatch, missing element, text invisible on background, etc.), (5) apply targeted Dart fixes and hot-reload via `flutter-wright`, (6) loop until visual delta is acceptable or 5 iterations hit, (7) always call `flutter-wright` reset before exit. Critically: do not transform or re-flatten the upstream design data; if mockplus-context's spec is insufficient that is mockplus-context's bug, not this skill's responsibility.
